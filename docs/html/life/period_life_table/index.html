Period life table

Warning

This module is in draft version.

This Excel add-in allows calculations on period life tables.

Conventions and notations

The following conventions and notations will be used throughout the rest of the documentation:

  • All time units are expressed in years.

  • The current age of an individual is denoted \(x\) (e.g., \(x=50\) for a 50 year old individual or \(x=50.75\) for an individual aged 50 years and 9 months).

  • The guaranteed interest rate is yearly compounded and denoted \(i\) (e.g., \(i=1.75\%\)).

Probabilistic view

The life expectancy of an individual can be studied from a probabilistic point of view as a random variable. The random variable describing the time-to-death (from birth) is denoted \(T\). Therefore, the remaining lifetime of an individual aged \(x\) is given by the random variable:

\[T_x = T-x \mid T>x,\]

that is the remaining lifetime (i.e. \(T-x\)) given that the individual is still alive at age \(x\) (i.e. knowing that \(T>x\)).

The life tables recognized by the module are listed below:

These tables are calculated using the Makeham model. The Makeham model is a parametric model assuming that the number of people alive at age \(x\) is given by

\[l_x = k s^x g^{c^x}\]

where \(k>0\), \(s \in (0,1]\), \(g \in (0,1)\) and \(c>1\). The following parameters are used to build the Belgian regulatory tables:

The alternative FK’ table is defined in the same way as the FK table except that the parameter \(c\) is set to 1.122 (leading to an increase in mortality). The unisex tables XR and XK are calculated for each integer age \(x\) as an arithmetic average of the MR-FR and MK-FK tables i.e.

\[l_x^{XR} = \frac{l_x^{MR} + l_x^{FR}}{2} \qquad \text{and} \qquad l_x^{XK} = \frac{l_x^{MK} + l_x^{FK}}{2}.\]

Note that all of these tables are only calculated for integer ages \(x\) (not for fractional ages) and that the number of people alive \(l_x\) is rounded to the nearest integer. In the case of fractional ages, the module assumes a constant force of mortality between two consecutive integer ages. Finally, the ultimate age is always less than or equal to 120 years.

Age correction

All the functions implemented in this module provide a possibility of age correction. An age correction increases or decreases the actual age of an individual. For instance, an age correction of -3 years assumes that an individual aged \(x\) (real age) is aged \(x-3\) instead (corrected age) 1. Age corrections are used to provide a safety loading when performing actuarial calculations. It is common practice to use a negative age correction (reduction) when calculating the present value of an annuity with the MR/FR/XR tables.

1

In practice, the corrected age is calculated as \(\max(x + correction, 0)\) to avoid negative ages when the correction is negative.
